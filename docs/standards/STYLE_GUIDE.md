# DarkSage Documentation Style Guide
**Version:** 1.0.0

## Voice
Clear, precise, calm, technical, and explainable. Avoid hype in engineering specifications.

## Headings
One H1 per Markdown document. Use numbered H2/H3 sections for controlled Codex volumes.

## Typography
Preferred publication fonts:
- Titles/Headings: Aptos Display or Aptos
- Body: Aptos
- Code: Consolas

## Terminology
Use the official term first; Codex-themed aliases may be presented as UI labels.
- Sage — AI assistant/intelligence layer
- Observatory — Market Data
- Watchtower — Scanner
- Forge — Strategy Builder
- Chronicle — Backtesting
- Guardian — Risk Engine
- Treasury — Portfolio

## Numbers and Dates
Use ISO dates (`YYYY-MM-DD`) in controlled metadata. State timezone when time matters.

## Requirements
Do not combine multiple independently testable behaviors into one requirement.
