# How to Write a DarkSage Codex Document

1. Copy the appropriate template.
2. Assign a unique ID before drafting.
3. Define purpose, scope, audience, and authority.
4. Separate timeless principles from implementation details.
5. Write normative requirements with `shall`.
6. Add acceptance criteria and edge cases.
7. Link related requirements and ADRs.
8. Mark unresolved items explicitly; do not guess.
9. Run technical and editorial review.
10. Change status only through the documented approval workflow.
11. Update traceability and changelog entries when approved.
